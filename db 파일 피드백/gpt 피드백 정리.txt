==================================================
1. 공통 반복 오류
==================================================

1) N.C. 과다 판정
- 발생 형태:
  - 불완전 보이싱을 즉시 N.C.로 처리
  - shell voicing를 즉시 N.C.로 처리
  - sustain / tail / decay / continuation 구간을 즉시 N.C.로 처리
  - sounding bass 또는 structural tone이 남아 있는데 N.C.로 처리
- 위배 규칙:
  - bass_carry_over_before_root_relabel
  - duration_occupancy
  - hc_analysis_mode_accompaniment_only_labeling
  - hc_initial_label_lock_forbidden
  - hc_inertia_override_requires_explicit_rejection_reason

2) upper-structure 후보의 final label 과확정
- 발생 형태:
  - dim / aug / altered / fragment를 candidate가 아닌 final label로 확정
  - 상성부 순간 형태를 bass-aware final label보다 우선 채택
  - upper-structure 모양이 실제 sounding bass를 지우는 방향으로 출력됨
- 위배 규칙:
  - hc_upper_structure_candidate_not_final_authority
  - hc_no_identity_erasing_simplification
  - hc_no_slash_erasing_simplification_at_output
  - slash_preservation_over_clean_simplification
  - slash_label_bass_awareness_distinction

3) same-root family 우선 실패
- 발생 형태:
  - same-root shell이 가능한데 alternate-root diminished/minor/slash를 먼저 채택
  - direct same-root family보다 파생 해석을 primary로 채택
  - observed structural tones가 same-root를 지지하는데 alternate reading으로 이동
- 위배 규칙:
  - bass_root_alignment
  - observed_pitch_set_integrity
  - hc_mandatory_per_segment_pipeline
  - hc_reanalysis_from_bass_on_new_root_event
  - design_philosophy: prefer direct same-root family before promoting an alternate upper-structure-derived label

4) bare slash 우선 실패
- 발생 형태:
  - bare slash가 기본값이어야 하는 sonority를 즉시 기능화
  - non-chord bass를 chord quality 내부로 조기 흡수
  - upper triad/shell over bass를 literal slash보다 tension/dominant 계열로 먼저 출력
- 위배 규칙:
  - literal_slash_before_same_root_no3_tension
  - non_chord_bass_defaults_to_bare_slash
  - hc_non_chord_bass_defaults_to_bare_slash
  - hc_prefer_literal_slash_over_same_root_no3_tension
  - hc_slash_label_is_not_bass_violation

5) 관측되지 않은 7th / tension 승격
- 발생 형태:
  - accompaniment에 직접 관측되지 않은 7th를 final label에 포함
  - accompaniment에 직접 관측되지 않은 maj7/add9/9/11/13을 final label에 포함
  - brief / off-beat / decorative extension을 structural extension으로 승격
- 위배 규칙:
  - structural_extension_promotion
  - hc_analysis_mode_no_unobserved_7th_or_tension_promotion
  - hc_no_extension_overpromotion_from_same_root_recovery
  - do not promote weak ornamental tones into final tension text

6) melody contamination
- 발생 형태:
  - melody 음이 missing 3rd/7th/tension 보충 근거로 사용됨
  - accompaniment만으로는 성립하지 않는 quality/extension이 final label에 반영됨
- 위배 규칙:
  - analysis_mode_accompaniment_only_labeling
  - hc_analysis_mode_accompaniment_only_labeling
  - strict separation between harmony-analysis mode and generation mode

7) carry-over / overlap / humanized timing 처리 실패
- 발생 형태:
  - 새로운 bass attack 없이 fresh root-position chord로 relabel
  - overlap / anticipation / delayed release를 구조적 chord change로 판정
  - carried bass를 유지하지 않고 새 root로 재해석
- 위배 규칙:
  - bass_carry_over_before_root_relabel
  - humanized_timing_tolerance
  - hc_no_root_relabel_before_new_bass_attack
  - hc_pre_output_bass_refresh_check_required
  - distinguish expressive timing from structural harmonic timing

8) no-3rd ambiguity 처리 실패
- 발생 형태:
  - 3도가 없는 sonority에서 quality를 조기 확정
  - structural 4th가 있는데 sus4 검토 없이 minor7/half-diminished/alternate-root로 이동
  - slash/sus 검토보다 convenient family를 먼저 채택
- 위배 규칙:
  - quality_undecidable_without_3rd_requires_sus4_check
  - sus4_quality_discrimination
  - sus4_branch_lock_before_alternate_minor7
  - hc_quality_undecidable_without_3rd_requires_sus4_check
  - hc_sus4_branch_lock_before_alternate_minor7
  - hc_sus4_over_false_minor_shell

9) 과분할
- 발생 형태:
  - 한 family 내부 redistribution을 별도 harmonic segment로 분할
  - arpeggio traversal 내부 lowest-note 변화만으로 새 chord 분할
  - maj9 -> maj7 -> triad 식의 표면 차이를 별도 segment로 과세분
  - tail / local decoration을 독립 harmony로 분할
- 위배 규칙:
  - texture_mode_detection_before_split
  - arpeggio_harmonic_bass_disambiguation
  - terminal_arpeggio_bass_exception
  - hc_arpeggio_bass_change_not_sufficient
  - hc_no_arpeggio_terminal_bass_overpromotion
  - hc_mandatory_split_audit_on_new_bass_event
  - hc_mid_measure_harmony_split_required
  - hc_segment_local_label_scope

10) 절차 집행 불완전
- 발생 형태:
  - bass 확인 전 upper-structure pattern matching으로 진입
  - candidate 단계와 final output 단계가 분리되지 않음
  - pre-output 검증 없이 slash 삭제 또는 root simplification 발생
  - 의미 있는 new-bass event 이후 segment-local reanalysis가 생략됨
- 위배 규칙:
  - mandatory_per_segment_pipeline_enforcement
  - execution_guardrails_pre_output_gate
  - hc_mandatory_per_segment_pipeline
  - hc_initial_label_lock_forbidden
  - hc_inertia_override_requires_explicit_rejection_reason
  - hc_reanalysis_from_bass_on_new_root_event
  - hc_no_retroactive_bar_wide_label_after_late_confirmation

2. 개별 부가 오류
==================================================

A) key estimate 오판
- 발생 형태:
  - 전역 key_estimate가 실제 중심과 어긋남
  - 이후 enharmonic naming, roman numeral, diatonic reading, borrowed reading이 연쇄적으로 흔들림
- 관련 규칙:
  - meter_verification
  - roman_numeral_clarity
  - estimated key center plus local tonicization
  - enharmonic resolution by harmonic context

B) harmonic_event / DB 저장 필드 누락
- 누락 유형:
  - bass_aware_label 누락
  - roman_numeral 누락
  - beat_start / beat_end 누락
  - literal_pitch_collection 누락
  - pitch_classes 누락
  - contains_root / third / fifth / seventh 누락
  - structural_extensions / surface_extensions 분리 저장 누락
  - retrieval tag 축 부족
- 관련 규칙:
  - db_storage_schema
  - keep three views distinct: literal sounding set / upper-structure / final bass-aware selected label

C) section / form 과확정
- 발생 형태:
  - verse / prechorus / chorus / bridge / AABA 등을 과도하게 강제 추정
  - low-confidence 상태에서 unknown / ambiguous 처리가 없음
- 관련 규칙:
  - song-form-aware section analysis
  - confidence and secondary tags
  - allow unknown / ambiguous section labels
