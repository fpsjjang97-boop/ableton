[2] 이번 검증이 "지수 요청"과 얼마나 일치하는가

거의 같은 취지로 검증했지만, 아래 caveat 는 있다.

2-1. 지수 요청과 일치한 부분

- 최신 git 수정 반영된 코드 사용
- inference_server 경로 사용
- active_lora='sft' 확인
- 기존 LoRA 체크포인트 재사용
- same class of sanity test (drums / strings) 재실행
- per_bar_note_starts / empty_bar_ratio / longest_empty_run / VVV gate 확인
- 출력 MIDI 저장 및 청취 가능 상태로 보관

2-2. 완전히 동일 재현이 아닌 부분

- OS 레벨에서 "python -m midigpt.inference_server" 를
  독립 프로세스로 오래 유지하는 방식은 Windows 환경에서 불안정해서,
  같은 FastAPI app 을 한 프로세스 안에서 uvicorn 으로 띄우는 방식으로 우회함
  (경로는 동일하게 /status, /generate_json 사용)

- "이전에 돌린 exact same sanity input / exact same seed" 는
  저장된 스크립트가 명확히 남아 있지 않고,
  server API 에 seed 인자가 노출되어 있지 않아 100% 동일 조건 고정은 불가했음

즉:
"같은 server code path로 재측정"은 맞지만,
"완전히 동일 seed 재현"까지 했다고 말하면 안 된다.
