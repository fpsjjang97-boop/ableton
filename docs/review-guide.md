# MIDI 검수 가이드라인

## 개요

이 문서는 MIDI 데이터를 검수하고, 원본-변주 쌍을 생성하는 작업자를 위한 가이드입니다.
검수된 데이터는 AI 변주 모델(Transformer) 학습에 사용됩니다.

> 목표: **200쌍 이상의 고품질 원본-변주 페어 데이터 확보**

---

## 작업 환경

| 항목 | 권장 도구 |
|------|----------|
| MIDI 편집 | Ableton Live, MuseScore, LMMS, FL Studio |
| MIDI 확인 | https://www.aisongmaker.io/ko/midi-editor |
| 코드 분석 | Hookpad, Chordify, 또는 직접 청음 |
| 파일 저장 | `reviewed/` 디렉토리에 정해진 형식으로 |

---

## 작업 흐름

```
1. MIDI 파일 선택 (Ableton/midi_raw/ 또는 외부)
        ↓
2. 재생하며 품질 확인
        ↓
3. 프레이즈 분리 (마디 단위)
        ↓
4. 코드 라벨링
        ↓
5. 구조/스타일 태깅
        ↓
6. 품질 평가 (1~5점)
        ↓
7. 3점 이상이면 → 변주 페어 생성
        ↓
8. JSON 메타데이터 작성
        ↓
9. reviewed/ 디렉토리에 저장
```

## 1. 프레이즈 분리

MIDI를 **의미 있는 단위**로 나눕니다.

### 분리 기준
- 보통 **4마디 또는 8마디** 단위
- 쉼표가 길거나 분위기가 바뀌는 지점에서 분리
- 너무 짧으면(2마디 미만) 합치고, 너무 길면(16마디 초과) 나눔

### 파일명 규칙
```
{곡ID}_phrase_{구간번호}.mid

예시:
schubert02_phrase_01.mid    (마디 1~8)
schubert02_phrase_02.mid    (마디 9~16)
schubert02_phrase_03.mid    (마디 17~24)
```

### 분리 방법
1. DAW에서 MIDI 열기
2. 프레이즈 구간 선택
3. "Export Selection as MIDI" 또는 해당 구간만 잘라서 저장

---

## 2. 코드 라벨링

각 프레이즈의 **코드 진행**을 기록합니다.

### 표기법
- 영어 코드 표기 사용: `C`, `Dm`, `G7`, `Bbm`, `F#dim` 등
- 마디별로 구분: `Bbm | Db | Ab | Eb`
- 전위(inversion)는 슬래시로: `C/E`, `G/B`
- 모르는 구간은 `?`로 표기 (나중에 다른 사람이 채움)

### 예시
```
bars: 1-4
chords: Bbm | Db | Ab | Eb

bars: 5-8
chords: Gbmaj7 | Db/F | Ebm7 | Bb7
```

## 3. 구조 태깅

프레이즈가 곡에서 어떤 역할인지 태깅합니다.

### 구조 태그 목록

| 태그 | 설명 | 특징 |
|------|------|------|
| `intro` | 인트로 | 주제 도입 전, 보통 조용함 |
| `theme_A` | A 주제 | 메인 멜로디 첫 등장 |
| `theme_B` | B 주제 | 대비되는 두 번째 멜로디 |
| `development` | 전개부 | 주제를 변형/발전 |
| `bridge` | 브릿지 | A와 B를 연결하는 경과구 |
| `climax` | 클라이맥스 | 가장 강렬한 부분 |
| `cadence` | 종지 | 프레이즈 마무리 |
| `coda` | 코다 | 곡의 마무리 |
| `transition` | 전환 | 구간 사이 연결 |

## 4. 스타일 태깅

해당 프레이즈의 **음악적 느낌**을 태깅합니다. **복수 선택 가능.**

### 스타일 태그 목록

| 카테고리 | 태그 |
|----------|------|
| **분위기** | `서정적`, `격렬`, `미니멀`, `웅장`, `몽환적`, `어두운`, `밝은`, `긴장감` |
| **텍스처** | `멜로디중심`, `화음중심`, `아르페지오`, `옥타브`, `유니슨`, `폴리포닉` |
| **리듬** | `느린`, `빠른`, `당김음`, `3연음`, `루바토`, `일정한` |
| **다이나믹** | `pp`, `p`, `mp`, `mf`, `f`, `ff`, `크레센도`, `디크레센도` |
| **기법** | `트릴`, `장식음`, `페달`, `스타카토`, `레가토`, `글리산도` |

## 5. 품질 평가

| 점수 | 기준 | 처리 |
|------|------|------|
| **5** | 완벽 — 음정/리듬/다이나믹 모두 정확, 음악적으로 훌륭 | DB 포함 + 변주 페어 생성 |
| **4** | 좋음 — 사소한 문제만 있음 (미세한 타이밍 등) | DB 포함 + 변주 페어 생성 |
| **3** | 보통 — 사용 가능하나 아쉬운 부분 있음 | DB 포함 (변주 페어는 선택) |
| **2** | 나쁨 — 음정/리듬 오류 다수 | **DB 제외** |
| **1** | 사용 불가 — 심각한 문제 | **DB 제외 + 삭제** |

### 주요 감점 기준
- 음정 오류 (분명히 틀린 음): -1점
- 리듬 오류 (박자가 안 맞음): -1점
- 벨로시티 이상 (전부 같은 세기, 또는 비정상적): -0.5점
- 잡음/노이즈 노트: -0.5점
- 불완전한 프레이즈 (중간에 잘림): -1점

## 6. 변주 페어 생성

**가장 중요한 작업입니다.** 원본 프레이즈를 직접 변주해서 페어를 만듭니다.

### 변주 유형별 가이드

#### 6-1. 리듬 변주 (`rhythm`)
- **피치는 그대로**, 리듬만 변경
- 예: 4분음표 → 8분음표 2개, 긴 음 → 당김음

```
원본:  ♩ ♩ ♩ ♩     (4분음표 4개)
변주:  ♪♪ ♩ ♪♪ ♩   (8분+4분 교차)
```

#### 6-2. 멜로디 변주 (`melody`)
- **리듬은 유사하게**, 피치를 변경
- 스케일 안에서 음을 바꾸거나, 경과음/장식음 추가

```
원본:  도 미 솔 도
변주:  도 레미 솔 라솔  (경과음 추가)
```

#### 6-3. 화성 변주 (`harmony`)
- 멜로디 유지, **반주/화음 구조** 변경
- 단순 화음 → 아르페지오, 넓은 보이싱 → 좁은 보이싱

```
원본:  [도미솔] 블록 화음
변주:  도-미-솔-도 아르페지오
```

#### 6-4. 다이나믹 변주 (`dynamics`)
- 음/리듬은 유사하게, **벨로시티/강약** 변경
- pp → ff 크레센도, 악센트 위치 변경

#### 6-5. 장식 변주 (`ornament`)
- 트릴, 턴, 모르덴트, 아포지아투라 등 장식음 추가
- 원본 멜로디의 뼈대는 유지

#### 6-6. 복합 변주 (`mixed`)
- 위 유형 2개 이상 동시 적용
- 예: 리듬 변주 + 화성 변주

### 변주 시 주의사항
- **원본의 조성(키)은 반드시 유지**
- 원본 멜로디의 **핵심 음(강박 위치의 음)은 보존**
- 변주가 너무 달라지면 "편곡"이 됨 — 원본과의 연관성 유지
- 자연스럽게 들려야 함 — 기계적 변환 지양

│   ├── schubert02_phrase_01_var_rhythm.mid

## 10. FAQ

**Q: MIDI 편집 도구가 없으면?**
A: 무료 도구 사용 가능 — MuseScore(악보 기반), LMMS(DAW), Online MIDI Editor

**Q: 코드를 모르겠으면?**
A: `?`로 표기하고 넘어가세요. 다른 작업자가 채울 수 있습니다. 코드보다 프레이즈 분리와 변주 생성이 더 중요합니다.

**Q: 변주를 얼마나 다르게 해야 하나?**
A: 원본을 듣고 "아, 이건 저 곡의 변주구나"라고 알 수 있는 수준. 원곡 멜로디의 강박 음이 50% 이상 유지되면 OK.

**Q: 한 프레이즈에 변주를 몇 개 만들어야 하나?**
A: 최소 1개, 권장 2~3개 (다른 변주 유형으로). 하나를 잘 만드는 게 여러 개 대충보다 낫습니다.

**Q: 외부 MIDI(DB에 없는 곡)도 추가 가능?**
A: 가능합니다. 저작권 문제 없는 MIDI(CC0, Public Domain, 직접 제작)만 사용하세요.
