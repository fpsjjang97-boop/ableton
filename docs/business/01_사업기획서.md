## 0. Executive Summary

**MidiGPT는 작곡가의 DAW 안에 거주하는, 음악 이론을 이해하는 자체 LLM이다.**

- 서비스: MIDI 입력 → 음악 이론(코드/스케일/송폼)을 알고 변주·편곡·확장을 만들어 DAW로 즉시 사용
- 차별화: 텍스트 LLM 래퍼가 아닌 **MIDI 전용 50M decoder-only 자체 모델 + 화성 마스킹 + Cubase 15 표현력 통합**
- 형태: VST3/CLAP 플러그인 + 데스크톱 워크스테이션 + LoRA 스타일 마켓플레이스
- 모델: 50M 베이스 + 스타일별 LoRA 핫스왑 (재즈·시티팝·메탈·클래식 등)
- 현재 단계: Phase 1(인퍼런스 안정화) 완료, Phase 2(데이터 수집) 진행 중. 제품화 이전.

## 1. 정체성 (Identity)

### 1-1. 우리는 누구인가
**"DAW 안에서 작곡가와 같이 호흡하는 AI 동료"** 를 만든다. 작곡가가 띄워놓은 곡을 보고, 코드 진행과 송폼을 이해하고, 그 결과를 **이미 작곡가가 쓰던 DAW의 트랙·아티큘레이션·CC·MPE 어휘** 그대로 출력한다.

### 1-2. 우리가 아닌 것
| 우리가 아닌 것 | 이유 |
|-----------------|------|
| ChatGPT 음악 래퍼 | 음악 이론을 모르고, MIDI 토큰 없이 텍스트 묘사만 함 |
| 오디오 생성 모델 (Suno/Udio) | MIDI를 못 만들어서 DAW 워크플로우에 못 들어감, 편집 불가능 |
| Magenta 데모 | 연구 결과물, 제품 아님, DAW 통합 0 |
| 단순 코드 자동 생성기 | 우리는 송폼/표현/CC까지 다룸 |

### 1-3. 신념
- **데이터가 모델이다.** 50M 파라미터에 잘 큐레이션된 데이터가 1750억 파라미터에 잡음 데이터보다 낫다.
- **DAW를 떠나지 말 것.** 작곡가가 마우스로 새 도구를 배워야 한다면 패배.
- **음악 이론은 마스킹으로 강제하라.** 추론 시 화성 마스킹 → 절대 off-scale 음 안 나옴. 학습이 부족해도 음악 규칙은 지킨다.
- **버그는 개발자가 잡지만, 품질은 작곡가가 정의한다.** 자동 리뷰어는 보조, 사람 청취가 정답.

## 2. 시장 정의

### 2-1. 1차 타겟 (Beachhead)
- **세미프로 작곡가** (Cubase / Ableton / Logic 사용자)
- 월 1회 이상 곡 작업, 막힐 때 빠른 변주·편곡 영감을 원하는 사람
- 한국 + 일본 + 동남아 약 30~50만 명 추정

### 2-2. 2차 타겟
- **게임/광고 BGM 작곡가** — 짧은 곡을 다량으로 변주해야 하는 직업군
- **영상 편집자/유튜버** — 저작권 안전한 BGM 변주가 필요
- **음악 학원/이론 교육** — 화성/송폼 시각화 도구로

### 2-3. 안 가는 시장
- 보컬 합성 (Suno/Udio 영역)
- 마스터링 (이미 LANDR 등이 점유)
- 라이브 퍼포먼스 (실시간 latency 요구가 다름, 별도 제품)

## 3. 제품 (Product Surface)

### 3-1. 기둥 4개
1. **MidiGPT 50M 자체 LLM** — 변주/편곡/확장 생성
2. **DAW 통합** — VST3/CLAP 플러그인, Ableton Bridge, Cubase 15 어휘
3. **Audio2MIDI / Sheet2MIDI** — 작곡가의 기존 자산(오디오·악보)을 LLM이 다룰 수 있는 MIDI로 변환
4. **LoRA 스타일 어댑터** — 재즈/시티팝/메탈/클래식 등 핫스왑

### 3-2. 단일 가치 제안
> **"내가 만들던 곡에 빠져서 막혔을 때, DAW 안에서 한 클릭으로 30초 변주 3개가 나오고, 그 중 하나를 그 자리에서 편집할 수 있다."**

## 4. 비즈니스 모델

### 4-1. 수익 구조 (단계별)
**Phase A: Pre-launch (현재 ~ 6개월)**
- 수익 0. 데모 + 동업자 음악가 패널 + 클로즈드 베타.

**Phase B: Public Beta (6-12개월)**
- 무료 데스크톱 앱 + LoRA 어댑터 1-2개 무료 + 추가 LoRA 유료 ($3-5/each, 일회성)

**Phase C: Commercial (12개월+)**
- **Pro 구독** $15/월 — VST3 플러그인, 무제한 변주, 모든 LoRA, 클라우드 학습 크레딧
- **Studio 라이선스** $200/year — 5석, 상업 사용
- **API** — B2B (게임 회사 BGM 자동화)
- **LoRA 마켓플레이스** — 외부 작곡가가 자기 스타일 LoRA를 학습/판매, 30% 수수료

### 4-2. 단위 경제 (가설)
- 50M 모델 추론 비용: 로컬 (RTX 3060+ 보유 가정) → 추론 비용 0
- 클라우드 옵션: A100 1초당 ~$0.001 → 30초 변주 1개 = ~$0.03
- 학습 비용 (LoRA 1개): 4090 24시간 ≈ ~$15
- Pro 구독 LTV (24개월 가정): $360, CAC 목표 < $40

## 5. 경쟁 우위 (Moat)

### 5-1. 기술적 해자
1. **MIDI 전용 어휘 + 화성 마스킹** — 텍스트 LLM 래퍼는 따라할 수 없는 음악 이론 강제
2. **Cubase 15 표현력 통합 (32 articulation, CC, MPE-ready)** — 단순 노트가 아닌 실연주 표현
3. **계층적 REMI 토크나이저 (448 vocab)** — 14 트랙 카테고리로 어휘 폭발 방지
4. **자체 데이터셋 큐레이션** — 동업자 작곡가가 직접 만든 고품질 시드 데이터

### 5-2. 데이터 해자
- 동업자 패널의 청취 평가 데이터가 누적될수록 **DPO/RLHF의 사람 라벨**이 쌓임
- 외부에서 모방 불가능 (작곡가 합의·계약 필요)

### 5-3. DAW 정체성 해자
- **VST3/CLAP 호스팅 모듈** 완성 시 — DAW를 떠나지 않고 워크플로우에 녹아드는 유일한 음악 LLM 도구
- 작곡가가 DAW를 바꿀 의향은 0에 가까움 → 일단 들어가면 락인
