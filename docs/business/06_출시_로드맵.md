## 0. 로드맵 철학

본 로드맵은 다음 원칙을 따른다:

1. **게이트(Gate) 기반**: 각 단계의 종료 조건이 명확하고, 통과 못 하면 다음 단계로 넘어가지 않는다
2. **품질 우선**: 사업적으로 빠르게 가고 싶어도, 외부 작곡가 패널 7/10 점수를 못 받으면 출시하지 않는다
3. **데이터 우선**: 모든 모델 작업은 데이터 게이트 통과 후 진행
4. **회수 가능**: 각 게이트 미달 시 이전 단계로 돌아가는 절차 존재

---

## 1. 단계 개요

```
M0 (현재)
  ↓ 데이터 회수 + 회귀 안정화
M1: 데이터 1500곡 + 사전학습 안정화 (gap < 1.5)
  ↓
M2: SFT + LoRA 어댑터 3개
  ↓
M3: CFG 학습 + 사람 청취 패널 검증
  ↓
M4: VST3/CLAP 호스팅 알파
  ↓
M5: Closed Beta (외부 50명, NPS ≥ 30)
  ↓
M6: Public Beta + Pro 구독
  ↓
M7: 1.0 정식 출시
```

**미통과 시 조치**: 작업 항목 재실행. 다음 단계 진입 금지.

## 3. M1 — 데이터 1500곡 + 사전학습 안정화 (M0 후 4주)

### 목적
50M 모델에 충분한 학습 신호를 공급해, 일반화 능력의 기초를 쌓는다.

### 작업
| # | 작업 | 책임 | 노력 | 의존성 |
|---|------|------|------|--------|
| 1 | 자동 품질 필터 코드 (트랙수/박자/밀도/키) | 개발자 | M | M0 완료 |
| 2 | Lakh MIDI Clean 다운로드 + 필터 적용 | 개발자 | M | 1 |
| 3 | 1차 1000곡 통합 (Lakh 필터 통과분) | 개발자 | S | 2 |
| 4 | GiantMIDI-Piano 일부 통합 (피아노 200-500곡) | 개발자 | M | 1 |
| 5 | 동업자 신곡 추가 (월 50-100곡 페이스) | 작곡가 | L | — |
| 6 | 사전학습 재실행 (1500곡, EMA on) | 개발자 | M | 3, 4, 5 |
| 7 | train/val gap 측정 + 진단 | 개발자 | S | 6 |
| 8 | GitHub Actions CI (인코딩/setup_check/1-epoch) | 개발자 | M | — |

### 게이트 (M1 → M2)
- [ ] 학습 데이터 ≥ 1500 곡 원본
- [ ] 자동 품질 필터 통과율 ≥ 80% (외부 데이터)
- [ ] **train/val gap < 1.5** (현재 2.6)
- [ ] 생성 MIDI 평균 크기 ≥ 5KB
- [ ] 14개 트랙 카테고리 중 ≥ 5종이 생성 결과에 등장
- [ ] CI 파이프라인 동작 (커밋 → 자동 검증)

**미통과 시 조치**:
- gap 1.5 ~ 2.0: 데이터 1500 → 2000곡 추가 수집 → 재학습
- gap 2.0+: 모델 또는 데이터 품질 진단 필요. M0로 회귀.
