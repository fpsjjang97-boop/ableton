## 문서 작성 원칙

### 표준 양식 (BalloonFlow_Doc 준수)
- 각 명세는 **개요 → 구성 → 동작 규칙 → 파라미터 → 의존성 → 검증 항목** 순서
- **현재 구현됨** 항목은 명확히 표시 (✅ / 🔶 부분 / ❌ 미구현)
- 코드 위치는 `파일경로:라인` 포맷
- 모든 수치/한계는 명시 (예: vocab=448, block_size=2048)

### 호환성 표시
- 🟢 안전: drop-in 변경
- 🟡 주의: 설정 잘못 잡으면 품질 저하
- 🟠 재학습 필요
- 🔴 호환성 깨짐 (체크포인트/어댑터 폐기)
