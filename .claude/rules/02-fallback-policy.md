# Rule 02 — Fallback 정책

"값이 없을 때 무엇을 할 것인가" 는 **컴포넌트별로 다르다**. 잘못된 fallback은 에러를 조용히 삼켜 5차까지의 버그 대부분의 원인이 되었다.

## 2.1 세 가지 상태를 구분한다

| 상태 | 의미 | 기본 액션 |
|------|------|-----------|
| **명시적 값** | 사용자/데이터가 지정함 | 그대로 사용 |
| **unknown** | 지정되지 않음, 알 수 없음 | 명시적으로 `"other"` / `"unknown"` 태그 |
| **에러** | 지정됐지만 규약 위반 | 예외 + 경고 로그, skip 또는 중단 |

**핵심:** "MIDI 기본값 0" 은 *명시적 값이 아니다*. "지정되지 않음"에 해당한다.

---

## 2.2 학습 데이터 파이프라인 — 엄격 모드

**원칙:** 데이터 이상은 모델 품질을 조용히 망가뜨린다. 관대한 fallback 금지.

### _classify_track (encoder.py)

**의도:** MIDI 트랙을 14개 카테고리 중 하나로 분류.

**규약:**
1. **트랙 이름이 비어있지 않다** → 이름 기반 매치가 1순위.
2. **이름 매치 실패 + 프로그램 번호가 명시됨 (non-zero, 또는 외부 표기상 지정됨)** → 프로그램 기반 매치.
3. **이름이 있지만 매치 실패 + 프로그램 = 0 (기본값)** → **프로그램 fallback을 건너뛴다.** 왜냐하면 program=0은 "Piano"가 아니라 "미지정"일 가능성이 높기 때문 (5차 리포트: 테스터가 program을 지정하지 않았다고 명시).
4. **모든 매치 실패 + 노트가 있음** → 피치 레지스터 기반 fallback (bass / melody).
5. **모든 매치 실패 + 노트 없음** → `"other"`.

### 과거 위반
- `_classify_track` 가 program=0 을 Piano family 로 간주해 `accomp` 로 일괄 매핑 → **5차 accomp 쏠림의 근본 원인**.
- 4차까지 누적된 "everything is accomp" 증상도 같은 뿌리.

### 이름 매치 예외 (substring 충돌)
- `"bass"` 매치: `"brass"` 와 `"bassoon"` 을 **둘 다 제외**해야 함. (bassoon = woodwind)
- `"drum"` 매치는 `"bass"` 매치보다 먼저 (BASSDRUM 같은 복합어 처리).
- 단수/복수형 둘 다 매치: `"string"` 과 `"strings"` 모두 인식.

## 2.3 추론 (inference) — 관대 모드

**원칙:** 생성된 토큰 시퀀스가 규약을 완전히 지키지 않을 수 있다. 합리적으로 복구.

### 허용되는 관대함
- 알려지지 않은 토큰 → skip (`_tokens_to_notes` 의 unknown branch)
- 순서가 어긋난 Pos/Pitch/Vel/Dur → 불완전 노트는 버리되, 이후 노트는 계속 디코드
- EOS 조기 등장 → `min_new_tokens` 로 suppression (토큰 길이가 최소치 미만이면 EOS logit을 masking)

### 금지되는 관대함
- BOS/EOS 누락을 근거로 "모든 시퀀스에 BOS를 임의로 prepend" 같은 **시퀀스 재구성** 금지 (학습 분포와 어긋남)
- 잘못된 Bar 순서 → **정렬하지 말 것**. 절대 인덱스 기반 디코딩(`current_bar * ticks_per_bar`)이 유효성을 보장.

### 과거 위반
- EOS suppression 이 일부 경로에만 적용되어 조기 종료 발생 (2026-04-08 리포트).
- 테스트 스크립트 한 곳에서는 fix, 서버 경로는 미적용 — **한 정책은 한 곳에서만 구현, 호출부는 공유**.

## 2.4 사용자 입력 (DAW / HTTP) — 검증 후 관대

**원칙:** 사용자의 실수로 서버를 죽이지 않는다.

### 규약
- 요청 스키마 검증 (FastAPI pydantic) → 실패 시 400 에러 + 한글 메시지
- 파일 크기 / 트랙 수 제한 확인 → 초과 시 422
- 누락된 선택 파라미터 → 문서화된 기본값 사용 (단, **학습 파이프라인이 쓰는 값과 다르게** — 사용자 UX용 기본값은 "관대", 학습용은 "엄격")

---

## 2.5 공통 안티패턴 (모두 금지)

1. **"혹시 모르니까 기본값" — 검증 없이 except-pass 뒤에 0/""/None 주입**
2. **"일단 돌아가게" — unknown 값을 가장 흔한 카테고리로 집단화** (accomp 쏠림의 원인)
3. **"경고 없는 skip" — loader가 잘못된 데이터를 조용히 건너뛰면 학습이 어긋난 걸 알 수 없음**. skip은 항상 warn + count.
4. **"try/except 광범위" — 무엇이 잡히는지 모르는 except는 규약 위반.** 구체 예외만 catch.
