■ 원인 추정
  FP16 학습 중 gradient overflow → NaN 전파로 추정.
  다음 중 하나 이상이 원인일 가능성:
  (1) learning rate가 FP16 LoRA에 비해 너무 높음
  (2) gradient clipping 미적용 또는 clip 값이 너무 큼
  (3) 일부 SFT 페어 데이터에 극단적인 토큰 값 존재
  (4) LoRA 레이어의 초기화 값이 FP16 범위를 벗어남

■ 수정 제안
  - train_sft_lora.py 에 gradient clipping 추가
    (예: torch.nn.utils.clip_grad_norm_(params, max_norm=1.0))
  - GradScaler (FP16 자동 스케일링) 적용 검토
  - learning rate 낮추기 (현재 값 → 절반 또는 1/10)
  - 문제 배치 필터링: 토큰 값 범위 검증 로직 추가


================================================================
[생성 결과 청취 — 부분 개선 확인]
================================================================

SFT LoRA 학습이 실패했기 때문에 생성은 사실상
pre-training 모델만으로 이루어졌습니다.

■ 개선된 점 (긍정적 피드백)
  이전까지는 accomp 트랙에 모든 악기 정보가 뒤섞여
  MIDI 노트들이 완전히 난잡하게 표시됐는데,
  이번에는 accomp 트랙의 노트들이 정상적으로 보임.
  피아노 트랙처럼 깔끔하게 표시되는 수준으로 개선됨.
  → accomp 트랙 분류 버그 수정(커밋 29a3e87)의 효과로 보임.

■ 여전히 남은 문제
  (1) 중간 공백 구간 — 여전히 미해결
      생성된 MIDI 중간중간 노트가 없는 구간이 존재.

  (2) 노트 중복 찍힘 — 여전히 미해결
      accomp 트랙을 포함한 여러 트랙에서
      동일한 위치에 MIDI 노트가 2개씩 겹쳐서 생성됨.
      DAW(Cubase)에서 확인 시 노트가 두 겹으로 표시.
      → 디코더에서 동일 포지션에 노트를 중복 출력하는
        로직 버그로 추정.

SFT NaN 버그가 수정되면 다시 테스트할 준비가 되어 있습니다.
